# Section Library (섹션 라이브러리)

각 섹션은 독립적인 모듈이지만, 하나의 구매 판단이나 핵심 베네핏이 반드시 하나의 섹션으로 끝나야 한다는 뜻은 아닙니다. 제품 장르와 마케팅 목표에 따라 필요한 섹션을 선택하고 순서를 조정하되, 복잡한 내용은 `장면/결과 -> 질문 -> 메커니즘/상세 -> 증거 -> 구매 정보`처럼 여러 모바일 화면 단위 섹션으로 쪼갭니다.

## 이미지 수량 원칙

섹션별 이미지 표기는 상한이 아닙니다. 모든 신규 상세페이지는 1번 히어로와 마지막 정적 CTA/클로징을 generated `FULL_IMAGE`로 계획해야 합니다. 그 외에는 필요한 경우 `FULL_IMAGE`와 `HTML_MIXED` 지원 이미지를 충분히 사용하고, 이미지 장수는 구매 여정, 스토리 연결, 증거 밀도, 옵션/보관/리뷰/비교/FAQ 보강 필요성으로 결정합니다. 텍스트나 표만으로 충분한 중간 섹션은 이미지 없이 둘 수 있지만, 장수 제한이나 고정 비율 때문에 이미지를 줄이면 안 됩니다.

## HOOK (후킹)

**목적**: 스크롤을 멈추게 하고 감정적 연결 생성

**타입 선택**:
- **상황형**: "{일상 장면} 때, {불편} 때문에 불편하셨다면"
- **조건 확인형**: "구매 전, {핵심 조건}만 꼭 확인해 주세요!"
- **베네핏형**: "{행동}만 하면 {좋아지는 점}을 더 편하게"
- **불안 해소형**: "{구매 불안}이 걱정된다면 이 부분을 먼저 봐주세요"
- **숫자형**: 실제 근거가 있을 때만 "{검증된 수치}로 확인한 {베네핏}"

**작성 가이드**:
- 타겟의 핵심 욕구나 고민을 건드리는 한 문장
- 호기심을 유발하되 과장하지 않기
- 제품명은 아직 언급하지 않음
- 2-3초 안에 읽히는 길이

**예시**:
> "짐 들고 들어올 때, 문이 다시 닫혀 불편하셨다면"
> "구매 전, 우리 집 문에 자석이 붙는지만 확인해 주세요!"
> "발로 살짝 내리면 필요한 순간에 문이 고정돼요"

**이미지**: 필요 수량, 상한 없음 (텍스트 중심이어도 시각 증거가 필요하면 추가)

## HERO (히어로)

**목적**: 제품의 첫인상과 핵심 가치 전달

**구성 요소**:
- 브랜드 마크 (영문 + 한글)
- 제품 카테고리 or 시리즈명
- 제품명 (한글, 임팩트 있게)
- 핵심 가치 한 줄 (서브 헤드라인)
- 태그라인 (브랜드 철학 반영)

**작성 가이드**:
- 제품명은 감성적 네이밍 권장
- 핵심 가치는 기능이 아닌 결과/감정 중심
- 태그라인은 브랜드와 제품을 연결

**예시** (가상 브랜드: 글로우랩 GLOWLAB - 스킨케어):
```
GLOWLAB · 글로우랩
더마 사이언스 라인

밸런싱 세럼 에센스

피부 본연의 빛을 되찾는 시간

"과학으로 빛나다"
```

**히어로 이미지 유형**:

| 유형 | 설명 | 적합한 경우 |
|------|------|-------------|
| **제품 사진** | 제품 단독 또는 연출 촬영 | 제품 자체가 시각적으로 매력적일 때 |
| **히어로 영역 디자인** | 제품 + 텍스트가 조합된 웹 디자인 | 브랜드 메시지 강조, 프로모션 |
| **미니멀 웹포스터** | 감성적 비주얼 + 타이포그래피 | 브랜드 무드 전달, 고급스러운 느낌 |
| **이미지 오버레이 타이포그래피** | 배경 이미지 위에 텍스트 오버레이 | 강렬한 첫인상, 메시지 중심 |

**이미지**: 필요 수량, 상한 없음 (첫인상에 필요한 만큼 필수)

## SOCIAL PROOF (사회적 증거)

**목적**: 신뢰 구축, "나만 모르는 건가?" 심리 자극

**필수 수치** (3개 이상):
- 누적 판매량 (예: 15,000+)
- 재구매율 (예: 34%)
- 평균 평점 (예: 4.9)
- 리뷰 수 (예: 2,847개)
- 판매 기간 (예: 런칭 6개월)

**작성 가이드**:
- 숫자는 신뢰감 있는 수준으로 (너무 과장 X)
- 가능하면 실제 데이터 기반
- 수치 + 라벨 형식으로 간결하게

**이미지**: 필요 수량, 상한 없음 (인증/뱃지, 수상 내역)

## REVIEWS (리뷰)

**목적**: 후기 리듬을 통한 신뢰 구축, 구매 후 모습 상상

**작성 가이드**:
- 모든 상세페이지에 포함
- 최소 3개, 권장 4-5개
- 실제 리뷰가 있으면 의미를 바꾸지 않고 자연스럽게 정리
- 실제 리뷰가 없으면 교체용 더미 리뷰 카드로 구성하고 내부 로그에만 `REVIEW_PLACEHOLDER_REPLACE_REQUIRED` 표시
- 교체용 더미 리뷰에는 이름, 연령대, 지역, 날짜, 별점, 리뷰 수, 구매 수, `실제 구매자` 같은 구체 사실을 쓰지 않음
- 리뷰 섹션 헤드라인은 확인형으로 작성: 실제 리뷰가 있으면 `실제 사용자 후기를 확인해 보세요`, 교체용 카드면 `사용 후기를 확인해 보세요` 또는 `사용 후기로 확인할 포인트`
- `먼저 써본 사람이 말해요`, `먼저 써본 사람들의 이야기`, `써본 사람은 이렇게 말합니다`처럼 후기를 가장한 진행자 문구는 쓰지 않음
- 각 카드는 다른 구매 상황이나 베네핏 반영
- 구체적 상황 + 사용 후 편해진 점 포함
- "좋아요" 같은 추상적 표현 피하기
- visible copy에는 `더미 리뷰`, `실제 리뷰 없음`, `교체 예정`, `NEEDS_PROOF` 같은 경고 문구를 쓰지 않음

**리뷰 구조**:
1. 구매 전 상황/고민
2. 제품 사용 경험
3. 구체적 변화/결과
4. 감정적 만족

**예시** (교체용 더미 카드):
```
"처음 써도 사용 순서가 어렵지 않았어요.
필요한 구성이 함께 있어 따로 찾을 일이 줄었습니다."
```

**이미지**: 필요 수량, 상한 없음 (리뷰 캡처, 전후 비교, 사용자 사진) - 📍위치: **리뷰 사이**

## EMPATHY (공감)

**목적**: "이 브랜드가 나를 이해한다" 느낌 전달

**5-1. 페인포인트 리스트**

타겟이 공감할 수 있는 고민 4-6개 나열:
```
혹시 요즘 이런 생각 하셨나요?

• {고민 1}
• {고민 2}
• {고민 3}
• {고민 4}
```

**⚠️ 주의: `☐`, `□` 같은 체크박스 기호 사용 금지** → 코딩 시 SVG 아이콘으로 자동 변환됨

**5-2. 공감 메시지**
```
반복되면 은근히 신경 쓰이는 순간이 있습니다.

{소비자 상황}

그래서 {제품명}은 {소비자 베네핏}에 집중했습니다.
```

**이미지**: 필요 수량, 상한 없음 (공감 분위기 이미지) - 📍위치: **텍스트 전**

## DOUBT BREAKER (의심 해소)

**목적**: 타겟 고객이 가질 수 있는 의심과 우려를 선제적으로 해소

**언제 사용**:
- 가격이 높은 제품
- 효과를 입증해야 하는 제품 (건강기능식품, 화장품 등)
- 신생 브랜드
- 온라인 구매 시 불안감이 클 수 있는 제품

**의심 유형별 해소 전략**:

| 의심 유형 | 예시 의심 | 해소 방법 |
|-----------|----------|-----------|
| 품질 의심 | "정말 효과 있을까?" | 임상 테스트, 인증서, 성분 분석 |
| 가격 의심 | "너무 비싼 거 아닌가?" | 구성 가치, 포함 혜택, 옵션 영역의 현재 혜택 확인 안내 |
| 브랜드 의심 | "이 브랜드 믿을 수 있나?" | 브랜드 스토리, 수상 이력, 미디어 노출 |
| 적합성 의심 | "내 피부/체질에 맞을까?" | 성분 안전성, 테스트 결과, 반품 정책 |
| 지속성 의심 | "얼마나 오래 갈까?" | 내구성 테스트, 보증 기간, 유지 비용 |

**예시** (가상: 프리미엄 에센스):
```
아직 망설여지신다면

💭 "정말 효과가 있을까?"
3주간 임상 테스트 결과, 피험자 89%가
피부 결 개선을 체감했습니다. (△△피부과학연구원)

💭 "비싼 거 아닌가?"
1병으로 약 2개월 사용.
1일 비용 약 650원으로 피부과 시술 1회보다 경제적입니다.

💭 "내 피부에 맞을까?"
무향료·무색소·저자극 포뮬러.
민감성 피부 테스트 완료.
그래도 걱정되시면? → 14일 무료 체험 후 환불 가능
```

**이미지**: 필요 수량, 상한 없음 (인증서, 테스트 결과, 비교 차트) - 📍위치: 텍스트 후

## TARGET AUDIENCE (타겟)

**목적**: "이건 바로 나를 위한 제품이구나" 확신

**구성**: 4-6개 타겟 세그먼트

**각 세그먼트 구조**:
- 아이콘 (이모지)
- 대상 (구체적 상황 포함)
- 추천 이유 (이 제품이 주는 가치)

**예시** (가상: 업무용 가방):
```
이런 분들께 추천드립니다

💼 매일 노트북을 들고 출퇴근하시는 분
   가볍고 수납력 좋은 가방이 필요하신 분

✈️ 출장이 잦으신 분
   기내 반입 가능하고 정리가 쉬운 가방을 찾으시는 분

🎓 대학원생/취준생
   면접과 미팅에도 어울리는 깔끔한 디자인을 원하시는 분
```

**이미지**: 필요 수량, 상한 없음 (타겟 라이프스타일, 사용 장면) - 📍위치: **텍스트 전**

## STORY (스토리)

**목적**: 제품에 의미와 깊이 부여, 감성적 연결

**스토리 유형 선택**:
- **기원 스토리**: 제품/소재의 역사적 배경
- **브랜드 스토리**: 왜 이 제품을 만들게 되었는지
- **상징 스토리**: 제품이 상징하는 의미
- **장인 스토리**: 만드는 사람들의 이야기

**작성 가이드**:
- 감성적이면서도 신뢰감 있는 톤
- 제품과 고객의 연결고리 제시
- 너무 길지 않게, 핵심만 (150-250자)

**예시** (가상: 수제 도자기 브랜드 - 흙담):
```
THE STORY

흙담의 시작

"완벽하지 않아도 괜찮아."

10년 차 도예가 김○○ 작가는
기계로 만든 완벽한 그릇에 의문을 품었습니다.

손으로 빚은 미세한 비대칭,
가마 속에서 우연히 생긴 유약의 흐름.
그 '불완전함'에서 발견한 아름다움을
일상의 식탁으로 가져오고 싶었습니다.
```

**이미지**: 필요 수량, 상한 없음 (스토리 이미지, 원산지/배경, 제작 현장) - 📍위치: **텍스트 전** 또는 사이

## FOUNDER / TEAM (창업자/팀 소개)

**목적**: 창업자나 핵심 인물의 신뢰성을 통해 제품/브랜드 신뢰 구축

**언제 사용**:
- 크라우드펀딩 프로젝트
- 스타트업/신생 브랜드
- 전문성이 중요한 제품 (의료, 기술, 교육 등)
- 창업자의 스토리가 브랜드 핵심인 경우
- 장인/전문가가 만드는 제품

**신뢰 요소 유형**:

| 유형 | 예시 |
|------|------|
| 학력/자격 | 서울대 화학공학 박사, 피부과 전문의 |
| 경력 | 삼성전자 10년, 前 ○○기업 CTO |
| 수상/인증 | 레드닷 디자인 어워드, 특허 5건 |
| 실적 | 누적 펀딩 5억, 시리즈A 투자 유치 |
| 미디어 | MBC 출연, 조선일보 인터뷰 |

**예시 1** (가상: 기술 스타트업):
```
FOUNDER

"수면의 질이 삶의 질입니다"

박○○ · SlpTech 대표

서울대학교 바이오메디컬공학 박사
前 삼성전자 무선사업부 수석연구원 (10년)
수면 관련 특허 12건 보유
2023 CES Innovation Award 수상

"30대에 불면증을 겪으며 깨달았습니다.
좋은 수면은 선택이 아니라 생존입니다."
```

**예시 2** (가상: 장인 브랜드):
```
THE ARTISAN

30년 외길, 가죽 장인

정○○ 마스터

1994년 첫 가죽 가방을 만든 이후
오직 한 길만 걸어왔습니다.

에르메스 공방 연수 (2005)
대한민국 명장 지정 (2018)
청와대 외교 선물 제작 (2019)

"손끝에서 전해지는 온기,
```

## CTA (구매 유도)

**금지**:
- 상세페이지 디자인에는 버튼 UI를 넣지 않음
- `[옵션 선택]`처럼 버튼처럼 보이는 괄호형 요소, 둥근 사각형 버튼, 클릭 가능한 링크 버튼 금지
- 구매 버튼은 판매 채널의 구매 영역이 담당하므로 상세페이지 섹션 안에 만들지 않음

**예시**:
```
글로우랩 밸런싱 세럼 에센스

구성에 맞는 옵션을 선택하고
현재 혜택을 확인해 주세요.

30ml / 50ml 구성은 옵션 영역에서 확인

배송 혜택은 옵션 영역에서 확인할 수 있어요.
```
