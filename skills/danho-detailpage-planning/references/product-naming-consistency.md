# 제품 호칭 일관성 룰 (Product Naming Consistency)

> 한 상세페이지 안에서 제품을 부르는 호칭이 **줄어들었다 늘어났다** 하면 브랜드 인식이 흐트러진다.
> 이 가이드는 PLANNING 단계에서 호칭을 명시적으로 정의하고, 코딩·이미지 단계에서 그 표기를 일관 적용하기 위한 규칙이다.

---

## 0. 핵심 한 줄

**PLANNING.md에 "정식 풀네임 / 정식 단축형 / 브랜드 단독" 세 가지 호칭을 정의하고, 그 외 줄임말은 금지.**

---

## 1. 호칭 3계층 정의 (PLANNING 단계 필수)

PLANNING.md의 기본 정보 표 바로 아래 또는 별도 "제품 호칭" 블록에 다음을 명시한다.

```markdown
## 제품 호칭 (필수 — 본문 전체에 일관 적용)

| 계층 | 표기 | 사용처 |
|---|---|---|
| 정식 풀네임 | 알파 식기세척기 초벌 전용 실리콘 수세미 | 첫 등장, hero, CTA, alt 텍스트의 정식 소개 |
| 정식 단축형 | 알파 실리콘 수세미 | 본문 paragraph, 카드 텍스트, alt 텍스트 일상 호칭 |
| 브랜드 단독 | ALPHA / 알파 | 푸터 로고, 브랜드 그래픽, 단독 노출만. 본문 호칭 금지 |
```

### 1-1. 룰 요약

- ❌ **금지**: 본문에서 "알파"라고만 줄여 쓰기 (브랜드 단독은 로고/푸터에서만)
- ✅ **권장**: 본문 첫 등장은 풀네임, 두 번째 등장부터는 단축형
- ✅ **반복 호칭**: 단축형 ("알파 실리콘 수세미") 일관 사용

### 1-2. 왜 안 줄여 쓰는가

- "알파"만 단독으로 쓰면 **무슨 제품인지 즉시 인지 안 됨** (브랜드명만 보이고 카테고리가 없어짐)
- 페이지를 스크롤로 훑는 소비자는 매 섹션이 독립적 — 매 섹션마다 "이게 무엇인지" 호칭으로 즉답해야 함
- 검색 SEO/접근성 측면에서도 단축형이 더 유리 (브랜드명 + 카테고리 키워드 동시 포함)

## 2. 적용 위치별 가이드

### 2-1. HTML 본문 — 단축형 우선

```html
<!-- ❌ BAD -->
<p>알파를 같이 식세기에 넣어 살균합니다.</p>
<h3>알파를 고른 3가지 이유</h3>
<p>알파는 식세기 앞 단계에 특화된 도구입니다.</p>

<!-- ✅ GOOD -->
<p>알파 실리콘 수세미를 같이 식세기에 넣어 살균합니다.</p>
<h3>알파 실리콘 수세미를 고른 3가지 이유</h3>
<p>알파 실리콘 수세미는 식세기 앞 단계에 특화된 도구입니다.</p>
```

### 2-2. HTML 본문 — 풀네임 등장 시점

다음 위치에서는 정식 풀네임 사용 권장:
- hero 섹션의 제품 소개 줄 (예: `<p>알파 식기세척기 초벌 전용 실리콘 수세미</p>`)
- final product/result closing 섹션의 제품명 헤더 (예: `<h2><span class="block">알파 식기세척기</span><span class="block accent">초벌 전용 실리콘 수세미</span></h2>`)
- 첫 단락 처음 호칭 (이후부터 단축형으로)

### 2-3. 이미지 안 텍스트 (REPLACE 케이스)

이미지 생성 프롬프트에서도 동일 룰:
- hero/final-closing 배너의 메인 카피 안 제품명 → **풀네임**
- 보조 라벨/캡션 → **단축형** 또는 생략

### 2-4. `alt` 속성

alt도 본문의 일부 — 단축형 일관 사용.

```html
<!-- ❌ -->
<img alt="끓는 물에 담긴 알파">
<!-- ✅ -->
<img alt="끓는 물에 담긴 알파 실리콘 수세미">
```

### 2-5. 브랜드 단독 ("ALPHA" 또는 "알파") 사용처

- footer 브랜드 로고: `<div class="footer-brand">ALPHA</div>` ✅
- 로고 그래픽 (이미지 안): "ALPHA" 단독 OK ✅
- 본문 paragraph 또는 heading: ❌ 단독 사용 금지

## 3. PLANNING.md 양식 (필수 블록)

`output-format.md`의 기본 정보 표 직후에 다음 블록을 반드시 포함:

```markdown
## 제품 호칭 (본문 전체 일관 적용 — 필수)

- **정식 풀네임**: {예: 알파 식기세척기 초벌 전용 실리콘 수세미}
- **정식 단축형**: {예: 알파 실리콘 수세미}
- **브랜드 단독**: {예: ALPHA} (푸터·로고 전용, 본문 금지)

> ⚠️ 본문 카피 작성 시 단축형을 기본으로 사용. 줄여서 "알파"만 쓰는 표기 금지.
```

## 4. 코딩 단계 검증 grep

빌드 완료 후 다음 grep으로 단독 브랜드명 잔존 검증:

```bash
# {정식 단축형}에 포함되지 않은 단독 {브랜드명} 등장 찾기
# 예: 단축형 "알파 실리콘 수세미" / 브랜드 "알파"
grep "알파" build/*.html | grep -v "알파 실리콘 수세미\|알파 식기세척기 초벌 전용 실리콘 수세미"

# 일반화 패턴: BRAND={브랜드 단독}, FULL_OR_SHORT_TOKENS={풀네임/단축형의 핵심 키워드 OR-연결}
# grep "BRAND" build/*.html | grep -v "FULL_OR_SHORT_TOKENS"
```

출력이 비어 있어야 합격. 단, ALPHA(영문 대문자)는 푸터·로고에서 정상 사용이라 별도 제외.

---

## 5. 자동 검증을 위한 체크리스트

PLANNING 작성 후:
- [ ] 제품 호칭 3계층(풀네임/단축형/브랜드 단독)을 PLANNING.md에 명시했다
- [ ] 본문 카피에서 단축형이 일관되게 사용됐다
- [ ] 줄여서 "{브랜드 단독}만" 쓴 본문 표기가 없다

HTML 빌드 후:
- [ ] 위 grep 명령 실행 결과가 비어 있다 (또는 의도적인 푸터·로고 케이스만 남는다)
- [ ] alt 속성에서도 단축형 또는 풀네임 사용
- [ ] hero·final-closing에는 풀네임, 본문에는 단축형

---

## 6. 예외 — 의도적 단독 브랜드 사용

다음은 예외적으로 단독 브랜드명("ALPHA" 또는 "알파") 사용 가능:
- footer 브랜드 로고 (`.footer-brand`)
- 이미지 안에 그래픽 로고로 들어가는 경우
- "Made by ALPHA" 같은 별도 표기 코너
- 인용·후기에서 자연스러운 호칭 ("저는 알파만 써요" 같은 직접 인용)

위 외에는 일관성 룰 적용.
