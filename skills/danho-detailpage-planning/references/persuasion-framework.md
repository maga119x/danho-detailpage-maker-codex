# Multi-Step Persuasion Framework (다단계 설득 구조)

하나의 설득 포인트를 1~3단계로 분할하여 깊이 있는 설득 구조를 만듭니다. 단순히 "이 제품은 좋습니다"가 아니라, 왜 필요하고 → 무엇이고 → 어떤 변화가 있는지를 순차적으로 전달합니다.

## 왜 단계 분할이 필요한가?

| 문제 | 결과 |
|------|------|
| 하나의 설득 포인트 = 하나의 섹션 | 설득력 부족, 피상적 |
| "좋은 성분입니다" 한 줄로 끝 | 왜 좋은지, 어떻게 다른지 모름 |
| 기능 나열만 함 | 고객이 왜 신경 써야 하는지 모름 |

---

## 3단계 설득 프레임워크

각 핵심 설득 포인트(FEATURES, MATERIALS, EMPATHY 등)를 아래 3단계로 확장합니다:

```
┌─────────────────────────────────────────────────────────────┐
│  1단계: 문제 인식 (Problem Recognition)                      │
│  - 왜 이것이 필요한지                                        │
│  - 고객이 겪는 문제/불편                                     │
│  - 기존 해결책의 한계                                        │
├─────────────────────────────────────────────────────────────┤
│  2단계: 해결책 제시 (Solution)                               │
│  - 우리 제품/기능이 무엇인지                                 │
│  - 어떻게 작동하는지                                         │
│  - 왜 다른 것과 다른지                                       │
├─────────────────────────────────────────────────────────────┤
│  3단계: 결과/효과 (Outcome)                                  │
│  - 사용 후 어떤 변화가 있는지                                │
│  - 구체적인 수치/증거                                        │
│  - 감정적 만족                                               │
└─────────────────────────────────────────────────────────────┘
```

---

## 설득 포인트별 분할 예시

### FEATURES (기능/특징) 분할

```
[1단계 - 문제 인식] 섹션
"왜 일반 이어폰으론 안 될까요?"
→ 지하철 소음, 집중력 저하, 볼륨 높이면 청력 손상

[2단계 - 기능 설명] 섹션
"액티브 노이즈 캔슬링이란?"
→ 기술 원리, 차별점, 특허/인증

[3단계 - 결과/효과] 섹션
"지하철에서도 음악에 더 집중할 수 있어요"
→ 사용자 체험담, 전후 비교, 검증된 수치 증거
```

---

### MATERIALS (소재) 분할

```
[1단계 - 문제 인식] 섹션
"왜 소재가 중요할까요?"
→ 저렴한 소재의 문제점, 피부 트러블, 내구성 이슈

[2단계 - 소재 설명] 섹션
"프랑스 노르망디산 프리미엄 린넨"
→ 원산지, 특성, 제조 과정

[3단계 - 결과/효과] 섹션
"10년 후에도 처음처럼"
→ 실제 착용감, 세탁 후 변화, 고객 후기
```

---

### EMPATHY (공감) 분할

```
[1단계 - 공감] 섹션
"혹시 이런 고민 있으셨나요?"
→ 페인포인트 체크리스트

[2단계 - 이해] 섹션
"반복되면 은근히 신경 쓰이는 순간이 있어요"
→ 고객의 실제 상황을 자연스럽게 대변

[3단계 - 희망] 섹션
"이제 {간단한 행동}으로 {베네핏}을 만들 수 있어요"
→ 해결 가능성 제시, 기대감 유발
```

---

### DOUBT BREAKER (의심 해소) 분할

```
[1단계 - 의심 대변] 섹션
"이런 걱정, 당연합니다"
→ 고객의 의심을 인정하고 공감

[2단계 - 근거 제시] 섹션
"그래서 준비했습니다"
→ 인증서, 테스트 결과, 데이터

[3단계 - 안심 보장] 섹션
"그래도 걱정되시면"
→ 환불 정책, 보증, 고객 지원
```

## 단계 분할 적용 가이드

| 설득 포인트 | 최소 단계 | 권장 단계 | 분할 우선순위 |
|-------------|-----------|-----------|---------------|
| FEATURES (핵심 기능) | 2단계 | 3단계 | ⭐⭐⭐ 높음 |
| MATERIALS (소재) | 2단계 | 3단계 | ⭐⭐⭐ 높음 |
| EMPATHY (공감) | 2단계 | 3단계 | ⭐⭐ 중간 |
| DOUBT BREAKER (의심 해소) | 2단계 | 3단계 | ⭐⭐⭐ 높음 |
| STORY (스토리) | 1단계 | 2단계 | ⭐ 낮음 |
| CRAFTSMANSHIP (제작 철학) | 1단계 | 2단계 | ⭐⭐ 중간 |
| REVIEWS (리뷰) | 1단계 | 1단계 | - |
| FAQ (자주 묻는 질문) | 1단계 | 1단계 | - |

---

## 실전 예시: 좋은 예 vs 나쁜 예

### 나쁜 예 (6개 섹션)

```
HOOK → HERO → FEATURES → MATERIALS → REVIEWS → FINAL-CLOSING
```
→ 각 포인트가 얕게 끝남, 설득력 부족

---

### 좋은 예 (15개 섹션)

```
HOOK → HERO → SOCIAL PROOF
→ EMPATHY-1(페인포인트) → EMPATHY-2(이해) → EMPATHY-3(희망)
→ MATERIALS-1(왜 성분이 중요한가) → MATERIALS-2(우리 성분) → MATERIALS-3(효과)
→ DOUBT BREAKER-1(의심 인정) → DOUBT BREAKER-2(근거) → DOUBT BREAKER-3(보장)
→ REVIEWS → TARGET → FAQ → FINAL-CLOSING → SPECS → FOOTER
```
→ 각 포인트를 깊이 있게 다룸, 설득력 높음

## 섹션 네이밍 규칙

단계 분할 시 섹션 이름은 다음과 같이 명명합니다:

```
[설득포인트]-[단계번호]: [핵심 메시지]

예시:
MATERIALS-1: 왜 성분이 중요한가
MATERIALS-2: 히알루론산이 필요한 이유
MATERIALS-3: 2주 후 달라진 피부

DOUBT BREAKER-1: 이런 걱정 하셨죠?
DOUBT BREAKER-2: 임상 테스트 결과
DOUBT BREAKER-3: 14일 무조건 환불
```

이렇게 분할하면 기획서에서 섹션 구조를 한눈에 파악할 수 있고, 코딩 스킬에서도 섹션 순서를 명확하게 이해할 수 있습니다.
