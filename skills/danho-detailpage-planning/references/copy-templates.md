# Korean Copywriting Templates

## Hook Section Templates

### Fit Check Hook (구매 조건 확인형)
```
구매 전, 이 부분만 꼭 확인해 주세요!
{condition}라면 {benefit}을 바로 사용할 수 있어요.
```

### Daily Moment Hook (생활 장면형)
```
{daily_moment} 때,
{pain_or_inconvenience} 때문에 불편하셨다면
{simple_outcome}을 더 쉽게 만들 수 있어요.
```

### Benefit Hook (베네핏형)
```
{buyer_action}만 하면
{desired_outcome}을 더 편하게 만들 수 있어요.
```

## Review Templates

Every detail page needs a review/testimonial section. Use supplied real reviews when available. If none are supplied, use replacement-ready dummy review cards and mark them only in planning/proof logs as `REVIEW_PLACEHOLDER_REPLACE_REQUIRED`.

Do not invent names, stars, ages, demographics, dates, review counts, purchase counts, or claims such as `실제 구매자`.

### Review Headline Rule

Use a check-oriented headline. Do not make the headline sound like a staged reviewer is speaking before the review cards.

- Supplied real reviews: `실제 사용자 후기를 확인해 보세요`, `구매 전, 실제 사용 후기를 살펴보세요`
- Replacement-ready placeholders: `사용 후기를 확인해 보세요`, `사용 후기로 확인할 포인트`
- Avoid: `먼저 써본 사람이 말해요`, `먼저 써본 사람들의 이야기`, `써본 사람은 이렇게 말합니다`

### Supplied Review Rewrite
```
실제 사용자 후기를 확인해 보세요

{supplied_review_excerpt_rewritten_naturally}
— {supplied_reviewer_label}
```

### Replacement-Ready Dummy Reviews
```
사용 후기를 확인해 보세요

{benefit_based_neutral_review_1}
{benefit_based_neutral_review_2}
{benefit_based_neutral_review_3}
```

## FAQ Templates

### Concern-Reassurance Format
```
Q: {concern_question}
A: {direct_answer}. {proof_point}. {additional_reassurance}.
```

### Common Questions
- 품질/진품 여부: "정품/천연 {material} 맞나요?"
- 사이즈/핏: "{size} 맞을까요?"
- 관리/내구성: "쉽게 {problem} 되지 않나요?"
- 선물 포장: "선물용 포장 되나요?"
- 교환/반품: "마음에 안 들면 반품 가능한가요?"

## Final Closing Templates

Final closing copy is not a CTA. The shopping mall UI already provides purchase buttons, option selectors, and benefit displays. End the detail page with product/result confidence, a use scene, brand tone, or quiet reassurance.

Do not include `구매하기`, `옵션 확인`, `혜택 확인`, `지금 보기`, `구성 확인`, `장바구니`, `주문`, bracketed button-like labels, or option/order-area prompts.

### Result Closing
```
{product_name}
{buyer_daily_result}이 조금 더 편해집니다.
```

### Scene Closing
```
{repeated_situation}마다
{product_core_result}을 자연스럽게 느낄 수 있어요.
```

### Brand Tone Closing
```
{brand_or_product_name}
{quiet_reassurance_or_philosophy}
```

## Doubt Breaker Templates

### Standard Format
```
💭 "{고객의 의심/우려}"
{해소 방법 + supplied 객관적 근거}
[supplied 수치/뱃지]
```

### Comparison Format
```
💭 "{가격/가치 의심}"
{비교 대상}과 비교하면
- {비교 포인트 1}
- {비교 포인트 2}
{buyer_decision_summary}
```

### Guarantee Format
```
💭 "{적합성/품질 의심}"
{안전성/테스트 결과}
그래도 걱정되시면? → {보장 정책}
```

### Crowdfunding Specific
```
💭 "배송이 늦어지지 않을까?"
현재 {supplied 진행 단계}입니다.
{supplied 배송 일정}에 맞춰 안내합니다.

💭 "스타트업이라 A/S가 걱정돼요"
{supplied 보증 기간} A/S 기준을 확인해 주세요.
고객센터 {supplied 운영 정보}
```

## Genre-Specific Phrases

### 뷰티/스킨케어
- "아침에 만졌을 때 편안한 피부감"
- "성분과 사용감으로 확인하는 변화"
- "민감성 피부도 안심"
- "트러블 없이 순하게"

### 건강기능식품
- "매일 챙기기 쉬운 루틴"
- "원료와 함량을 먼저 확인해 주세요"
- "바쁜 날에도 간단하게"
- "섭취 기준을 지켜 꾸준히"

### 패션/의류
- "입었을 때 바로 느껴지는 핏"
- "편안함과 스타일, 둘 다"
- "세탁해도 변하지 않는 핏"
- "매일 입고 싶은 옷"

### 가전/테크
- "복잡한 설정 없이 바로 사용"
- "손이 덜 가는 일상"
- "자주 쓰는 기능은 더 간단하게"
- "구매 전 호환 조건을 먼저 확인"

### 크라우드펀딩
- "함께 만들어가는 제품"
- "첫 번째 서포터가 되어주세요"
- "필요한 제품을 함께 완성해 주세요"
- "아이디어에서 현실로"

### 장인/핸드메이드
- "손끝에서 전해지는 온기"
- "시간이 만들어낸 가치"
- "기계로는 불가능한 것들"
- "느리지만 오래가는"

## 이미지 제안 템플릿

### 히어로 이미지 유형별 제안

#### 제품 사진 (Photo)
```
📷 제품 사진 제안
- 유형: 제품 단독샷 / 연출샷 / 라이프스타일
- 배경: {배경 설명}
- 앵글: {촬영 각도}
- 조명: {조명 유형}
- 분위기: {무드 키워드}

🎨 Codex 네이티브 이미지 생성 필요 시: `danho-imageprompt-helper` 스킬 호출
```

#### 히어로 영역 디자인 (Hero Design)
```
🖼️ 히어로 영역 디자인 제안
- 유형: e-commerce hero design
- 레이아웃: {분할 구조 - 예: 좌측 제품, 우측 텍스트}
- 텍스트 영역:
  - 헤드라인: "{한글 헤드라인}"
  - 서브카피: "{한글 서브카피}"
- 컬러 스킴: {색상 조합}
- 무드: {브랜드 톤앤매너}

🎨 Codex 네이티브 이미지 생성 필요 시: `danho-imageprompt-helper` 스킬 호출 (디자인 유형)
```

#### 미니멀 웹포스터 (Minimal Web Poster)
```
🎨 미니멀 웹포스터 제안
- 유형: minimalist web poster design
- 비주얼: {핵심 비주얼 요소}
- 타이포그래피:
  - 메인 카피: "{한글 메인 카피}"
  - 폰트 스타일: {고딕/명조/임팩트}
- 컬러: {메인 컬러 + 포인트 컬러}
- 여백: {화이트 스페이스 활용 방식}

🎨 Codex 네이티브 이미지 생성 필요 시: `danho-imageprompt-helper` 스킬 호출 (디자인 유형)
```

#### 이미지 오버레이 타이포그래피 (Image Overlay Typography)
```
✨ 이미지 오버레이 타이포그래피 제안
- 유형: full-bleed background with text overlay
- 배경 이미지: {배경 사진/그래픽 설명}
- 오버레이 처리: {그라데이션/블러/다크닝}
- 텍스트:
  - 헤드라인: "{강렬한 한글 메시지}"
  - 위치: {center/top/bottom}
  - 크기: {extra-large/large}
  - 폰트: {bold sans-serif/elegant serif}
- 대비: {텍스트 가독성을 위한 처리}

🎨 Codex 네이티브 이미지 생성 필요 시: `danho-imageprompt-helper` 스킬 호출 (디자인 유형 - 오버레이형)
```

### 이미지 유형 선택 가이드

| 제품 특성 | 권장 이미지 유형 |
|----------|-----------------|
| 제품 자체가 시각적으로 매력적 | 제품 사진 |
| 브랜드 메시지/프로모션 강조 | 히어로 영역 디자인 |
| 감성적 브랜드 무드 전달 | 미니멀 웹포스터 |
| 강렬한 첫인상/메시지 중심 | 이미지 오버레이 타이포그래피 |
| 고급스러운 느낌 | 미니멀 웹포스터 / 오버레이 타이포그래피 |
| 정보 전달 중심 | 히어로 영역 디자인 |
