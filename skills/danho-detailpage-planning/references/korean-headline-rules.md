# 한국어 헤드라인 카피 규칙 (Korean Headline Copy Rules)

> 한국 상세페이지·랜딩페이지·광고 카피에서 **헤드라인 끝에 마침표(.)는 찍지 않는다.**
> 영문 카피 관례를 그대로 옮겨 마침표를 찍으면 카피의 호흡과 임팩트가 죽고, 한국 광고/이커머스 카피 관례와 어긋난다.

---

## 0. 핵심 한 줄

**헤드라인·슬로건·CTA·섹션 제목 끝에 마침표(.)를 찍지 말 것. 본문(설명·문장) 끝에만 마침표.**

---

## 1. 헤드라인 끝 부호 룰

### 1-1. 금지 (NEVER)

- ❌ 헤드라인 끝 **마침표(.)**
- ❌ 부제(sub-headline) 끝 **마침표** — 부제도 카피의 일부
- ❌ CTA 문구/옵션 cue 끝 **마침표**
- ❌ 섹션 제목(h2/h3) 끝 **마침표**
- ❌ 슬로건/태그라인 끝 **마침표**
- ❌ 배지(badge) 라벨 끝 **마침표**

### 1-2. 허용 (OK)

- ✅ 질문형 끝 **물음표(?)** — "왜 초벌은 남을까요?"
- ✅ 강한 단언 끝 **느낌표(!)** — "지금 만나보세요!" (단, 남발하지 말 것)
- ✅ 한 헤드라인을 두 줄로 끊는 **쉼표(,)** — "장모로 훑고, / 단모로 밀어냅니다"
- ✅ 줄바꿈으로만 호흡 — 끝 부호 자체를 안 쓰는 게 가장 자연스러움
- ✅ 본문/설명 paragraph 끝 **마침표** — 정상

### 1-3. 케이스별 예시

| 카피 유형 | ❌ 나쁜 예 | ✅ 좋은 예 |
|---|---|---|
| 단언형 슬로건 | "초벌이 가벼워집니다." | "초벌이 가벼워집니다" |
| 두 줄 헤드라인 | "장모로 훑고, / 단모로 밀어냅니다." | "장모로 훑고, / 단모로 밀어냅니다" |
| 가치 제안 | "두 가지 면." | "두 가지 면" |
| 섹션 제목 | "사용 후기." | "사용 후기" |
| CTA 안내 | "옵션은 단순하게." | "옵션은 단순하게" |
| 결론 톤 | "오래오래." | "오래오래" |
| FAQ 헤더 | "자주 묻는 질문." | "자주 묻는 질문" |
| 질문형 hook | "왜 초벌은 남을까요?" | (그대로 OK — ?는 허용) |

---

## 2. 왜 안 쓰는가 (논거)

### 2-1. 카피의 호흡

마침표는 **문장의 종결**을 알리는 부호다. 한국어 헤드라인은 한 줄로 끊어 던지는 것이라, 마침표가 들어가면 카피가 "닫혀버린다." 다음 줄로 시선이 흐르지 않고 멈추게 된다.

### 2-2. 광고 카피의 관례

신문·잡지 광고부터 디지털 광고까지, 한국 광고 카피는 **헤드라인에 마침표를 거의 쓰지 않는다.** 카피라이팅 교본(예: 박웅현, 김민준, 송숙희 등)에서도 헤드라인은 "여백으로 닫고 마침표로 닫지 않는다"는 원칙을 공유한다.

### 2-3. 시각적 흐름

여러 섹션이 이어지는 상세페이지에서 헤드라인마다 마침표가 들어가면 페이지 전체가 답답하게 닫혀 보인다. 마침표 없이 줄바꿈만으로 마무리하면 다음 섹션으로 자연스럽게 시선이 이동한다.

---

## 3. 본문은 다르다

이 규칙은 **헤드라인·슬로건·짧은 카피**에만 적용된다.

- ✅ `<p>` 본문 paragraph 끝 마침표 — 정상
- ✅ FAQ 답변(`Q. ~? / A. ~합니다.`) — 답변은 일반 문장이므로 마침표 정상
- ✅ 리뷰 인용("실제로 써보니 좋아요.") — 일반 문장
- ✅ 배송·교환 안내 ("평일 오후 2시 이전 주문 시 당일 출고합니다.") — 정보형 문장

판정 기준: **이 카피가 "여운"으로 닫혀야 하는가, "정보"로 닫혀야 하는가**.
여운 → 마침표 X. 정보 → 마침표 O.

## 4. HTML 매핑 (코딩 단계 참고)

| HTML 요소 | 마침표 사용 |
|---|---|
| `<h1 class="hero-headline">` | ❌ 금지 |
| `<h2 class="section-title">` | ❌ 금지 |
| `<h3 class="section-subtitle">` | ❌ 금지 (카드 안 짧은 제목) |
| `<span class="badge">` | ❌ 금지 |
| `.cta-cue`, `.option-cue` 같은 정적 CTA 문구 | ❌ 금지 |
| `<p class="section-intro">` (짧은 도입 한 줄) | ❌ 금지 |
| `<p class="section-desc">` (1줄짜리 슬로건성) | 케이스별. 마무리 슬로건이면 ❌ |
| `<p class="section-desc">` (2~3줄 설명) | ✅ 일반 문장이면 마침표 OK |
| `<p>` 카드 본문 (긴 paragraph) | ✅ 마침표 OK |
| `<li class="check-text">` | 짧은 라벨이면 ❌, 문장이면 ✅ |

> 모호한 케이스("이 한 줄이 슬로건인가 설명인가")는 **읽었을 때 여운으로 닫는 게 자연스러우면 마침표 X.**

## 5. 자가 검증 체크리스트

기획서 작성 후 또는 코딩 후 마지막으로 한 번 훑기:

- [ ] 모든 `h1`, `h2`, `h3` 끝에 `.` 가 없다
- [ ] CTA 문구/옵션 cue 끝에 `.` 가 없다
- [ ] 슬로건성 `section-desc` 끝에 `.` 가 없다 (e.g. "단순하게.", "오래오래.")
- [ ] 부제(`section-intro`) 끝에 `.` 가 없다
- [ ] `<span class="block accent">`처럼 헤드라인의 두 번째 줄 끝에도 `.` 가 없다
- [ ] 단, 본문 paragraph와 FAQ 답변은 정상 마침표

### 빠른 grep 검증 (개발자용)

```bash
# v2.html에서 헤드라인성 요소가 마침표로 끝나는 경우 찾기
grep -E 'block accent">.*\.</span>' build/*.html
grep -E 'section-title">.*\.</' build/*.html
grep -E 'hero-headline">.*\.</' build/*.html
grep -E 'section-subtitle">.*\.</' build/*.html
grep -E '(cta-cue|option-cue)">.*\.</' build/*.html
```
출력이 비어 있어야 합격.

---

## 6. 예외

- **외래어 한글 표기** 끝 마침표 — 안 함 (예: "이미지 (이미지.)" X)
- **약어** 뒤 마침표 — 사용 가능 ("vs.")
- **시리즈 표기** ("1. hook 2. hero")의 번호 뒤 마침표 — 코드/리스트 문법이라 무관

---

## 7. 작성자 메모

- 이 규칙을 어기는 가장 흔한 이유: **AI 모델이 영문 헤드라인 관례를 한국어로 가져와서 마침표를 자동 삽입**. AI 카피 생성 시 이 가이드를 시스템 프롬프트 또는 검증 단계에서 강제할 것
- 다국어 페이지 운영 시 한국어판만 마침표 제거, 영문판은 그대로 두는 식의 다중 정책 가능
